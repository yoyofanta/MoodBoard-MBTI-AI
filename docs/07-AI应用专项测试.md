# AI 应用专项测试说明

## 一、测试目标

本专项测试主要验证系统中 AI 应用能力的稳定性和可用性，包括大模型接口调用、Prompt拼接、RAG检索增强、Memory用户记忆、Tool Calling工具调用和多Agent编排。

## 二、测试内容

### 1. 大模型接口测试

- mock模式下是否能返回固定回复
- real模式下是否能正常调用 DeepSeek
- API Key为空时是否有兜底提示
- 模型名错误时是否不会导致前端白屏
- AI返回为空时是否有默认提示

### 2. Prompt 测试

- 日常树洞 Prompt 是否偏温柔治愈
- INTJ Prompt 是否体现理性规划
- INFP Prompt 是否体现共情支持
- 多Agent Prompt 是否避免角色混乱
- 是否限制 AI 不输出无关内容

### 3. RAG 测试

- 是否能检索到相关知识片段
- TopK 数量是否正确
- 回答是否结合知识库内容
- 无相关知识时是否有兜底回答

### 4. Memory 测试

- 用户最近情绪是否写入
- 最近问题是否写入
- 最近 AI 回答是否写入
- 下一轮对话是否能读取 Memory
- 清空 Memory 是否生效

### 5. Tool Calling 测试

- 浏览器能否获取经纬度
- 后端能否调用高德接口
- 返回地址是否可展示
- 高德 Key 为空时是否提示明确

### 6. 多Agent测试

- 选择2个Agent能否开始
- 选择超过4个Agent是否拦截
- 每个Agent是否至少发言一次
- Agent回复是否体现不同人格风格
- 是否生成综合总结

## 三、测试结论

系统已具备基础 AI 应用能力，能够完成 RAG、Memory、Tool Calling 和多Agent编排等核心流程。异常场景下已增加兜底逻辑，能够避免 AI 接口异常导致前端不可用。
