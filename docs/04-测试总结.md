### BUG-001 缺陷说明

测试过程中发现，在情绪日记编辑页面点击“获取位置”后，页面只能显示“当前位置”，无法显示具体市区信息。通过浏览器 Network 分析发现，前端已成功获取经纬度，`/api/location/reverse` 接口也能正常返回 200，但响应数据中的 `province`、`city`、`district` 字段为空。

初步定位为后端高德地图 Web服务 Key 未正确配置，导致逆地理编码接口未成功返回地址信息。配置正确的高德 Web服务 Key 并重启后端后，重新执行获取位置测试，页面已能正常显示市区信息，缺陷已关闭。

## 九、测试结论

经过第一轮功能测试和核心接口测试，MoodBoard × MBTI 项目主要功能流程运行正常。用户可以完成登录、游客访问、情绪日记记录、位置获取、AI树洞聊天、MBTI人格对话、心灵盲盒和多人格对战等操作。

测试过程中发现的定位显示问题已完成修复，并通过回归测试验证。当前系统整体功能完整，核心流程稳定，基本满足短学期 Web 开发项目展示和后续测试岗位简历项目包装要求。

## 十、后续优化建议

1. 增加更多异常场景测试，例如 DeepSeek API 超时、网络断开、空内容提交、超长文本输入等。
2. 使用 Postman 或 Apifox 补充接口测试集合，便于后续重复执行。
3. 增加自动化接口测试脚本，例如使用 Python requests 对登录、日记、聊天接口进行批量验证。
4. 补充兼容性测试，包括不同浏览器、不同屏幕尺寸和手机端访问测试。
5. 优化 AI 人格 Prompt，使不同 MBTI 人格回复风格更加明显。
